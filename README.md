# Accessibility Toolkit
We want to create a toolkit to help open source projects understand, measure, and improve their accessibility by turning accessibility work into visible, trackable signals.

For now, this project has examples of useful files and templates to add to your open source projects:
- [ACCESSIBILITY.md](/ACCESSIBILITY.md): your accessibility statement and how to report issues
- [.github/ISSUE_TEMPLATE/accessibility.yml](/.github/ISSUE_TEMPLATE/accessibility.yml): accessibility bug reports
- [.github/PULL_REQUEST_TEMPLATE.md](/.github/PULL_REQUEST_TEMPLATE.md): include an a11y checklist
- `docs/accessibility/`: any project-specific guidance (component rules, patterns, known issues)
